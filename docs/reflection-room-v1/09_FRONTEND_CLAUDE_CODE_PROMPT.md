# Claude Code Prompt — Reflection Room V1 Frontend Implementation

> **Repo:** `mirror_collective_app` (frontend)
> **Companion:** `04_CLAUDE_CODE_PROMPT.md` is the backend equivalent for `mirror_collective_python_api`.

---

## How to use this

1. Open a terminal in the root of `mirror_collective_app`.
2. Confirm you're on the feature branch (e.g., `feat/reflection-room-v1`) — see `KICKOFF.md` for the branch setup.
3. Confirm the spec docs are in `docs/reflection-room-v1/` in this repo.
4. Run `claude` to start Claude Code.
5. Paste the prompt below in **The Prompt** as your first message.
6. Claude Code will explore the repo, propose Phase 0, then stop and wait. Reply `go` when ready.

---

## What to do if Claude Code goes off-script

- If it tries to invent UI copy: stop it, point at §12 of the UI handoff.
- If it tries to hardcode the 18 Mirror Moment button labels: stop it, point at §6.2 matrix.
- If it tries to fetch the snapshot more than once per journey: stop it, point at §1 caching rule.
- If it modifies an existing feature (MirrorGPT chat, archetype quiz, profile): stop it, that's the hard "do not touch" rule.
- If it asks about a content gap (motif why_texts, tone library lines, the 14 missing practices): direct it to use placeholder strings AND log a TODO referencing Tier 3 #16 / #17 / #18 in `05_GAPS_AND_OPEN_QUESTIONS.md`.

## After Claude Code finishes a phase

Pull up the diff. Spot-check:
- One Figma string against §12 verbatim.
- One Mirror Moment label generation through the matrix function.
- One loading / empty / error trio for the phase's screen.

Then say `continue` to start the next phase.
